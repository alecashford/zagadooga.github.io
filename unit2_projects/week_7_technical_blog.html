Week 7: Technical Blog - May 17, 2014

SQL injection is a serious security issue for all websites that include a database. Most websites store information in databases, which are simply repositories of raw information ordered very much like an excel spreadsheet. Much of this information is not going to be of interest to a hacker, but if he knows what he is looking for, he can “inject” lines of SQL code into input fields on your website and have returned to him personal information of the people registered on your website.

For example, a clever hacker could input an SQL query that returned all data in a given table, under the condition that 1 = 1. Because 1 = 1 is always going to be true, the website will return every field where that is true, which is everywhere.

It is possible to protect against SQL injection attacks by using SQL parameters, which will be evaluated and added to the server-side SQL in a controlled manner. SQL injection is intended to change the meaning of the original SQL code by modifying it so that it changes its function to a similar, valid, but harmful line of SQL. Using parameters prevents this because it treats the input as separate from the SQL code in the website and is not executed as a whole.
